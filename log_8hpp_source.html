34 /// @brief The logging levels.
35 /**
36 There are a few main levels (ordered from the less important
37 to the most important):
38  - hive::log::LEVEL_TRACE is used to log almost everything
39  - hive::log::LEVEL_DEBUG is used to log debug information
40  - hive::log::LEVEL_INFO is used to log main information
41  - hive::log::LEVEL_WARN is used to log potentially dangerous situations

43  - hive::log::LEVEL_FATAL is used to log fatal errors

531 /// @brief The "File" target.
532 /**
533 Sends log messages to the file stream.
534 
535 All write operations may be automatically flushed to disk but of course it
536 reduces the overal application performance. The "auto-flush" logging level
537 is used to control this process. By default it's hive::log::LEVEL_WARN,
538 so all **WARN**, **ERROR** and **FATAL** message will be automatically
539 flushed, other messages (**INFO**, **DEBUG**, **TRACE**) will not.
540 
541 A log file size limit might be specified as long as a maximum number
542 of log file backups. The file size limit isn't perfect precised and has the
543 log message boundary. That means that the whole log message will be written
544 although there is only one byte of free space in the log file. Once log file
545 is full the "backup" procedure will be started:
546 
547 | before | after | comments |
548 |------------|------------|----------------------------|
549 | `my.log.4` | | deleted (the oldest) |
550 | `my.log.3` | `my.log.4` | rename: `log.3` -> `log.4` |
551 | `my.log.2` | `my.log.3` | rename: `log.2` -> `log.3` |
552 | `my.log.1` | `my.log.2` | rename: `log.1` -> `log.2` |
553 | `my.log.0` | `my.log.1` | rename: `log.0` -> `log.1` |
554 | `my.log` | `my.log.0` | rename: `log` -> `log.0` |
555 | | `my.log` | created (the newest) |
556

558 
559 By default there is no any file size limit and therefore no any backups.

1695 Targets
1696 -------
1697 
1698 The log target is an endpoint for all log messages. hive::log::Target is the
1699 base class of all log targets. It also works like a **NULL** target -
1700 all messages just disappear.
1701 
1702 There are a few standard targets:
1703  - hive::log::Target::Stream sends log messages to an external stream like `std::cerr`.
1704  - hive::log::Target::File sends log messages to a text file.
1705  - hive::log::Target::Tie sends log messages to several child targets.
1706 
1707 It is easy to create new log target by overriding
1708 hive::log::Target::send() virtual method.
1709 
1710 Each target may be binded with custom message format through
1711 hive::log::Target::setFormat() which will be used for all
1712 log messages written to this target.
1713 
1714 It is possible to filter log messages by hive::log::Target::setMinimumLevel().
1715 All messages with logging level less than that limit will disappear (i.e. won't
1716 be written). Note, for hive::log::Target::Tie instances it's possible to
1717 specify minimum logging level for each child target individually.
1718 
1719 
1720 Formats
1721 -------
1722 
1723 How the log messages are written to the target is controlled by
1724 a hive::log::Format class instance. This instance is bound to
1725 the corresponding log target via hive::log::Target::setFormat() method.
1726 
1727 There is simple format which is used by default:
1728 ~~~
1729 timestamp logger level message
1730 ~~~
1731 
1732 But it is easy to customize formatting:
1733  - by overriding hive::log::Format::format() virtual method
1734  - or by using custom format string with the `printf`-like syntax.
1735 
1736 Have a look at the hive::log::Format class documentation for more details.

1739 Loggers
1740 -------
1741 
1742 Loggers are used to separate different log streams. For each logger it's
1743 possible to specify custom name, log target and logging level.
1744 
1745 It's very easy to create logger instances, just provide the custom name:
1746 
1747 ~~~{.cpp}
1748 hive::log::Logger a("/app/test/A");
1749 hive::log::Logger b("/app/test/B");
1750 ~~~
1751

1753 The root of hierarhy is special hive::log::Logger::root() logger instance
1754 which is also available via empty name.
1755 For the example above the logger hiearachy will be:
1756  - the root logger hive::log::Logger::root()
1757  - `"app"`
1758  - `"test"`
1759  - `"A"`
1760  - `"B"`
1761

1763 the hive::log::LEVEL_AS_PARENT logging level by default.
