Detailed Description

The "File" target.

Sends log messages to the file stream.

All write operations may be automatically flushed to disk but of course it reduces the overal application performance. The "auto-flush" logging level is used to control this process. By default it's hive::log::LEVEL_WARN, so all WARN, ERROR and FATAL message will be automatically flushed, other messages (INFO, DEBUG, TRACE) will not.

A log file size limit might be specified as long as a maximum number of log file backups. The file size limit isn't perfect precised and has the log message boundary. That means that the whole log message will be written although there is only one byte of free space in the log file. Once log file is full the "backup" procedure will be started:

before after comments
my.log.4 deleted (the oldest)
my.log.3 my.log.4 rename: log.3 -> log.4
my.log.2 my.log.3 rename: log.2 -> log.3
my.log.1 my.log.2 rename: log.1 -> log.2
my.log.0 my.log.1 rename: log.0 -> log.1
my.log my.log.0 rename: log -> log.0
my.log created (the newest)

Note, it's not recommended to provide log file size limit with no backups.

By default there is no any file size limit and therefore no any backups.

Log file is always open in append mode.
