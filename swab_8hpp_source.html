753 /** @page page_hive_swab Byte order tools
754 
755 There are two platform types: *little-endian* (LE) and *big-endian* (BE).
756 They are differ in way to save/load multibyte integers to/from memory.
757 For example, the hexadecimal `AABBCCDD` value (which is `2864434397` decimal)
758 stored in memory in the following ways:
759 
760 | # | LE | BE | |
761 |---|------|------|------------------------|
762 | 0 | `AA` | `DD` | lowest memory address |
763 | 1 | `BB` | `CC` | |
764 | 2 | `CC` | `BB` | |
765 | 3 | `DD` | `AA` | highest memory address |
766 
767 Most of Intel processors are little-endian. Some processors like ARM and MIPS
768 support both modes. In network protocols big-endian format is preferable.
769 
770 
771 Platform byte order macro
772 -------------------------
773 
774 This header file defines the following macro which might be used to byte-order
775 specific code:
776 - #HIVE_LITTLE_ENDIAN is defined on little-endian platforms
777 - #HIVE_BIG_ENDIAN is defined on big-endian platforms
778 - #HIVE_BYTE_ORDER is defined as `1234` or `4321` on little-endian
779  and big-endian platforms respectively.
780 
781 
782 Change the byte order
783 ---------------------
784 
785 There are a few functions to change the byte order:
786 - hive::misc::swab_16() for 16-bits integers
787 - hive::misc::swab_32() for 32-bits integers
788 - hive::misc::swab_64() for 64 bits integers
789 - hive::misc::swab() overloaded for all integer sizes
790 
791 The signed integers might be safely converted too.
792 
793 
794 Little/big-endian and host
795 --------------------------
796 
797 There are a few functions to convert little-endian and big-endian integers
798 to the host byte order.
799 
800 The following functions are used to convert little-endian integers
801 to the host byte order and visa versa:
802 - hive::misc::le2h_16(), hive::misc::h2le_16()
803 - hive::misc::le2h_32(), hive::misc::h2le_32()
804 - hive::misc::le2h_64(), hive::misc::h2le_64()
805 - hive::misc::le2h(), hive::misc::h2le()
806 
807 The following functions are used to convert big-endian integers
808 to the host byte order and visa versa:
809 - hive::misc::be2h_16(), hive::misc::h2be_16()
810 - hive::misc::be2h_32(), hive::misc::h2be_32()
811 - hive::misc::be2h_64(), hive::misc::h2be_64()
812 - hive::misc::be2h(), hive::misc::h2be()
813 
814 Both signed and unsigned integers may be converted.
815 
816 
817 Byte order converters
818 ---------------------
819 
820 The hive::misc::LittleEndian and hive::misc::BigEndian classes can be used as
821 little-endian and big-endian converters. See their static methods documentation.
822 */
